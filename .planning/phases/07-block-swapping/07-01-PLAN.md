**Hook algorithm (from RESEARCH.md):**

Forward pass (blocks 0..39 in order):
- `forward_pre_hook(block[i])`: Synchronize -- wait for any pending async transfer of block[i] to complete on the secondary stream. Call `torch.cuda.current_stream().wait_stream(self._stream)` to ensure main stream waits for the data.
- `forward_hook(block[i])`: If block[i] is in the swappable range, submit async transfer: move block[i] weights to CPU pinned buffers, and prefetch the next block that will be needed from CPU to GPU. Use `with torch.cuda.stream(self._stream)` context for async transfers.

Backward pass (blocks 39..0 in reverse):
- `full_backward_hook(block[i])`: After backward through block[i], if block[i] was brought to GPU for backward, submit async offload of block[i] to CPU. Also prefetch block[i-1] if it needs to come from CPU. Same async stream pattern.

**Critical implementation details:**
1. **Pinned memory**: Allocate pinned CPU buffers ONCE per swappable block at init. Use `torch.empty_like(param, device="cpu").pin_memory()` for each Linear weight in the block. Store in `self._pinned_buffers: dict[int, list[torch.Tensor]]`.
2. **Weight identification**: Only swap weights of modules whose class name ends with "Linear" (catches both regular `nn.Linear` and bitsandbytes `Linear8bitLt`/`Linear4bit`). Do NOT swap buffers, layer norm params, or non-Linear weights.
3. **CUDA synchronization**: In forward_pre_hook, call `torch.cuda.current_stream().wait_stream(self._stream)` to prevent the main stream from executing the block before its weights arrive. This prevents the NaN/illegal-memory-access pitfall from RESEARCH.md Pitfall 1.
4. **Move whole block to GPU first, then selectively offload Linear weights to CPU**: This ensures buffers (layer norm running stats) stay on GPU. Matches musubi approach.
5. **Hook handles**: Store all `RemovableHandle` objects in `self._handles: list` for cleanup via `remove_hooks()`.
6. **Logging**: Log at INFO level during __init__: "Block swap: {blocks_to_swap}/{num_blocks} blocks offloaded, {num_resident} resident on GPU". No per-step logging. Log a warning if blocks_to_swap > 35 (extreme swap count, significant speed impact).

**Helper methods:**
- `_get_linear_modules(block) -> list[tuple[nn.Module, str]]`: Walk block's modules, yield (module, param_name) where class name ends with "Linear" and module has a weight attribute.
- `_create_pinned_buffers(block, block_idx)`: Allocate pinned CPU tensor for each Linear weight in the block.
- `_move_block_to_device(blocks, block_idx, device)`: Move only Linear weights of block to target device using non_blocking=True within self._stream context.
- `_wait_for_block(block_idx)`: Synchronize the secondary stream transfer for this block.

Module-level docstring should explain: This module provides block-level CPU<->GPU weight swapping for training VRAM reduction. Based on the musubi-tuner Offloader algorithm but implemented via PyTorch hooks for compatibility with standard diffusers models.
