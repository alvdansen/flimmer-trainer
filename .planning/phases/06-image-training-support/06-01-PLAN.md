<objective>
Enable the encoding pipeline to handle image targets: VAE-encode single images via PIL (not ffmpeg), auto-assign self-referencing for I2V image samples, and thread the I2V context through the cache CLI so images work end-to-end in cache-latents.

Purpose: Without this, adding .jpg/.png files to training/targets/ causes ffmpeg errors during cache-latents, and I2V mode has no first-frame conditioning for image samples.

Output: Modified vae_encoder.py, expand.py, __main__.py with tests proving image encoding and auto self-referencing work.
</objective>

```python
class SampleRole(str, Enum):
    TARGET_VIDEO = "target_video"
    TARGET_IMAGE = "target_image"
    CAPTION = "caption"
    REFERENCE_IMAGE = "reference_image"

class DiscoveredSample(BaseModel):
    stem: str
    target: Path
    target_role: SampleRole
    caption: Path | None = None
    reference: Path | None = None
    width: int = 0
    height: int = 0
    frame_count: int = 0
    repeats: int = 1
    loss_multiplier: float = 1.0

class ExpandedSample(BaseModel):
    sample_id: str
    source_stem: str
    target: Path
    target_role: SampleRole
    caption: Path | None = None
    reference: Path | None = None
    bucket_width: int
    bucket_height: int
    bucket_frames: int
    frame_extraction: FrameExtraction = FrameExtraction.HEAD
    frame_offset: int = 0
    repeats: int = 1
    loss_multiplier: float = 1.0

    @property
    def is_image(self) -> bool:
        return self.bucket_frames == 1
```
